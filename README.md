## Reflection

In this project, I applied version control concepts using GitHub to manage a software project efficiently. I created a structured repository that included milestones and issues to simulate an Agile development workflow.

Milestones were used to define deadlines and track the progress of the first working version of the project. Issues were created to represent individual tasks such as adding the Python application file and the HTML template. Each issue was assigned and linked to the milestone, demonstrating task management and accountability.

One challenge I faced was understanding how to properly organize tasks and maintain clear documentation. However, by using GitHub features such as commits, issue tracking, and repository structure, I improved my ability to manage a project in a structured and professional way.

This activity helped me understand how version control systems support collaboration, project tracking, and iterative development in real-world software engineering environments.
